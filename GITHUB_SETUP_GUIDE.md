# GitHub Gist 동기화 설정 가이드

## ✅ 완료된 작업

Firebase를 GitHub Gist로 완전히 전환했습니다!

## 📋 사용 방법

### 1. GitHub Personal Access Token 생성

1. GitHub에 로그인
2. 우측 상단 프로필 클릭 → **Settings**
3. 왼쪽 메뉴에서 **Developer settings** 클릭
4. **Personal access tokens** → **Tokens (classic)** 클릭
5. **Generate new token (classic)** 클릭
6. Note에 "Soliloquy 동기화" 입력
7. Expiration 설정 (원하는 기간)
8. **Scopes**에서 **gist** 체크
9. **Generate token** 클릭
10. 생성된 토큰을 **복사** (한 번만 보여줌!)

### 2. 앱에서 토큰 입력

1. 앱 실행
2. 사용자 ID 입력 모달이 나타나면:
   - **사용자 ID** 입력
   - **GitHub Personal Access Token** 입력 (선택사항)
3. **확인** 클릭

### 3. 동기화 확인

- 메모를 작성하면 3초 후 자동으로 GitHub Gist에 저장됩니다
- 다른 기기에서 같은 사용자 ID와 토큰을 입력하면 자동으로 동기화됩니다
- 5초마다 자동으로 업데이트를 확인합니다

## 🎯 주요 기능

- ✅ **완전 무료** - GitHub Gist 무료 사용
- ✅ **무제한 동기화** - API 제한: 시간당 5,000회 (충분함)
- ✅ **비공개 Gist** - 데이터가 공개되지 않음
- ✅ **버전 관리** - Git 히스토리로 복원 가능
- ✅ **자동 동기화** - 5초마다 업데이트 확인

## ⚠️ 주의사항

1. **토큰 보안**: GitHub 토큰은 절대 공개하지 마세요!
2. **토큰 권한**: `gist` 권한만 있으면 됩니다
3. **동기화 지연**: 실시간이 아닌 5초마다 확인합니다 (폴링)

## 🔧 문제 해결

### 토큰이 유효하지 않습니다
- 토큰을 다시 생성하고 입력하세요
- `gist` 권한이 있는지 확인하세요

### 동기화가 안 됩니다
- 브라우저 콘솔에서 오류 메시지 확인
- 인터넷 연결 확인
- 토큰이 올바른지 확인

### Gist를 찾을 수 없습니다
- 다른 사용자의 Gist ID를 사용한 경우
- Gist가 삭제된 경우
- 새로 Gist를 생성하세요

## 📊 Firebase vs GitHub Gist

| 항목 | Firebase | GitHub Gist |
|------|----------|-------------|
| 비용 | 무료 (제한) | **완전 무료** ✅ |
| API 제한 | 일일 50,000 | 시간당 5,000 ✅ |
| 실시간 | ✅ | ❌ (5초 폴링) |
| 버전 관리 | ❌ | **✅ (Git)** |
| 프라이버시 | 서버 저장 | 비공개 Gist ✅ |
